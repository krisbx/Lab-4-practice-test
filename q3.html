Typesetting, Desktop Publishing, HTML, and CSS.
Old Fashion typesetting involved the fully manual process where an individual would need to press metal letters (or sorts) into a case backwards, place the tightly bound page of letters into ink, and stamp it onto a sheet of paper. This technique was extremely time consuming, as those doing it would need to have the ability to read backwards and have excellent precision. Typesetting was also very expensive, as any new font type, punctuation, or font size would require completely new sorts to be built.
Desktop Publishing (DTP) was created in the 1980s and introduced the use of software applications, a widely known one being PageMaker. DTP allows it's users to change the font, size, and position of its text. Users could also see what these changes looked like, unlike typesetting where you would need to print the page to view it. DTP started as a "print-publication" only idea since the introduction to online content was still in the future.
HTML (HyperText Markup Language) is a tool used for structuring elements on a web page. These elements can be headers, paragraphs, links, images, and more. HTML does not focus on the design and aesthetic of these elements. Unlike DTP, HTML allows the content it produces to be responsive to aspect ratios, so viewing the work on both a phone and a desktop will yield the same visual response. HTML also allows for quick and infinite modifications, with the changes being instantaneous in the output.
CSS (Cascading Stye Sheets) works together with HTML and provides the aesthetic on the web page. It controls various elements, some being colours, sizes, spacing, and placement. CSS allows for practically pixel-perfect adjustments that ensure its media provides responsive displays. The final fact about CSS is that it can be modified (commonly with JavaScript) to enable interactive elements that react to user input (buttons, sliders, etc.).
